## 一、系统架构优化

### 1.1 整体架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                         七步法人机协同系统                            │
├─────────────────────────────────────────────────────────────────────┤
│  用户层  │  决策确认 │ 创意输入 │ 质量审核 │ 最终批准                    │
├─────────────────────────────────────────────────────────────────────┤
│  交互层  │  对话界面 │ 命令接口 │ 可视化面板 │ 通知系统                  │
├─────────────────────────────────────────────────────────────────────┤
│  AI层   │  分析Agent │ 生成Agent │ 检查Agent │ 学习Agent                │
├─────────────────────────────────────────────────────────────────────┤
│  规则层  │  安全红线 │ 质量标准 │ 流程约束 │ 版本控制                    │
├─────────────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 人机分工矩阵

| 环节 | 人工职责 | AI职责 | 协同方式 |
|------|----------|--------|----------|
| **Step1 诊断** | 提供需求、确认目标 | 5W1H分析、SWOT生成、SMART建议 | 人工输入→AI分析→人工确认 |
| **Step2 蓝图** | 确认架构、调整范围 | WBS拆解、依赖分析、里程碑设定 | AI生成→人工评审→迭代优化 |
| **Step3 策略** | 选择路径、确认优先级 | 方案生成、风险评估、资源配置 | AI建议→人工决策→AI细化 |
| **Step4 资源** | 确认资源、风险评估 | 清单生成、成本估算、保障方案 | AI规划→人工确认→AI完善 |
| **Step5 执行** | 每日更新、问题上报 | 进度跟踪、风险预警、变更管理 | 人工记录→AI分析→建议推送 |
| **Step6 监控** | 异常处理、方向调整 | 实时监控、趋势分析、预警生成 | AI监控→预警→人工干预 |
| **Step7 复盘** | 经验提炼、知识归档 | 数据分析、模式识别、报告生成 | AI分析→人工总结→知识沉淀 |

## 二、状态流转优化

### 2.1 步骤状态机

```
                    ┌──────────┐
                    │  未开始  │
                    └────┬─────┘
                         │ 启动命令
                         ▼
┌──────────┐      ┌──────────┐      ┌──────────┐
│  已归档  │◄─────│  进行中  │─────►│  待审核  │
└──────────┘      └────┬─────┘      └────┬─────┘
      ▲                │                │
      │                │ 完成产出        │ 人工审核
      │                ▼                ▼
      │           ┌──────────┐      ┌──────────┐
      └───────────┤  已完成  │◄─────┤  审核中  │
                  └──────────┘      └──────────┘
```

### 2.2 详细状态定义

| 状态 | 编码 | 说明 | 转换条件 |
|------|------|------|----------|
| 未开始 | `PENDING` | 步骤未启动 | 接收启动命令 |
| 进行中 | `IN_PROGRESS` | AI正在处理 | 开始生成产出 |
| 待审核 | `PENDING_REVIEW` | 产出待人工确认 | AI完成初稿 |
| 审核中 | `UNDER_REVIEW` | 人工正在审核 | 人工打开产出 |
| 已完成 | `COMPLETED` | 审核通过 | 人工确认完成 |
| 已归档 | `ARCHIVED` | 已存入知识库 | 完成知识沉淀 |
| 已暂停 | `PAUSED` | 异常暂停 | 触发暂停条件 |
| 已终止 | `TERMINATED` | 流程终止 | 人工终止命令 |

### 2.3 状态流转规则

```yaml
# 状态流转规则配置
state_transitions:
  PENDING:
    - trigger: start
      target: IN_PROGRESS
      condition: 输入参数完整

  IN_PROGRESS:
    - trigger: ai_complete
      target: PENDING_REVIEW
      condition: AI产出完成且通过自检
      timeout: 300  # 5分钟超时

    - trigger: error
      target: PAUSED
      condition: 发生错误或异常

  PENDING_REVIEW:
    - trigger: human_review
      target: UNDER_REVIEW
      condition: 人工开始审核
      timeout: 1440  # 24小时提醒

    - trigger: request_revision
      target: IN_PROGRESS
      condition: 人工要求修改
      max_iterations: 5

  UNDER_REVIEW:
    - trigger: approve
      target: COMPLETED
      condition: 人工确认通过

    - trigger: reject
      target: IN_PROGRESS
      condition: 人工要求重做

  COMPLETED:
    - trigger: archive
      target: ARCHIVED
      condition: 完成知识沉淀

  PAUSED:
    - trigger: resume
      target: IN_PROGRESS
      condition: 问题解决

    - trigger: terminate
      target: TERMINATED
      condition: 人工终止
```

```yaml
quality_standards:
  # Step 1 诊断质量标准
  step1:

      - 每个预警有建议措施
      - 偏差必须有根因分析

  # Step 7 复盘质量标准

      - 成功经验必须提炼出可复用模式
      - 失败教训必须有根因分析
```

#### AI处理流程
```python
def step1_process(requirement, time_constraint, resource_constraint, **kwargs):
    """Step 1 详细处理流程"""

    # 1. 输入验证
    validate_input(requirement, min_length=10, max_length=500)

    # 2. 5W1H分析
    five_w_one_h = {
        "What": extract_what(requirement),
        "Why": infer_why(requirement),
        "Who": identify_stakeholders(requirement),
        "When": parse_time(time_constraint),
        "Where": infer_context(requirement),
        "How": suggest_approach(requirement)
    }

    # 3. SWOT分析
    swot = generate_swot(requirement, five_w_one_h)

    # 4. SMART目标生成
    smart_goals = generate_smart_goals(
        requirement,
        five_w_one_h,
        time_constraint,
        resource_constraint
    )

    # 5. 风险评估
    risks = identify_risks(swot)

    # 6. 生成报告
    report = compile_report(
        five_w_one_h=five_w_one_h,
        swot=swot,
        smart_goals=smart_goals,
        risks=risks
    )

    return report
```

### 4.6 Step 6 监控调整 - 操作细则

#### 监控指标体系
```yaml
monitoring_metrics:
  schedule:
    - metric: 计划完成率
      formula: 已完成任务数 / 计划任务数 × 100%
      threshold:
        warning: <80%
        critical: <60%

    - metric: 进度偏差(SV)
      formula: 挣值(EV) - 计划值(PV)
      threshold:
        warning: <-10%
        critical: <-20%

  cost:
    - metric: 成本偏差(CV)
      formula: 挣值(EV) - 实际成本(AC)
      threshold:
        warning: <-10%
        critical: <-20%

    - metric: 成本绩效指数(CPI)
      formula: EV / AC
      threshold:
        warning: <0.9
        critical: <0.8

  quality:
    - metric: 缺陷率
      formula: 缺陷数 / 产出物数量 × 100%
      threshold:
        warning: >5%
        critical: >10%

  risk:
    - metric: 风险暴露度
      formula: Σ(风险概率 × 风险影响)
      threshold:
        warning: >中等
        critical: >高
```
