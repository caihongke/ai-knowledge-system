# 项目复盘清单

> 适用于：未来所有使用七步法的项目
> 来源：AI 自学知识管理系统 MVP 项目复盘
> 生成时间：2026-03-21

---

## 启动阶段检查

- [ ] 问题/目标是否用 SMART 格式定义？（可衡量、有截止日期）
- [ ] 是否做了 SWOT 分析？（识别内外部因素）
- [ ] 差距分析是否量化？（现状 vs 目标的具体差距）
- [ ] 是否定义了优先级排序？（核心功能 > 锦上添花）

## 规划阶段检查

- [ ] WBS 是否拆解到可独立验收的任务粒度？（单个任务 ≤ 1 天）
- [ ] 每个工作包是否有明确的验收标准？
- [ ] 依赖关系是否画出来了？（无循环依赖）
- [ ] 关键路径是否识别？
- [ ] 里程碑是否设置了奖励机制？（保持动力）

## 策略阶段检查

- [ ] 每个工作包是否对比了至少 2 种方案？
- [ ] 是否为每个外部依赖定义了降级方案（Plan B）？  ⬅️ 重要！
- [ ] 技术选型是否基于 MVP 原则？（够用就好，不过度设计）

## 资源阶段检查

- [ ] 所有外部依赖是否已实际验证可用？  ⬅️ 本项目教训！
  - API Key 是否有效？
  - 第三方凭证是否过期？
  - 付费资源是否在预算内？
- [ ] 时间预算是否合理？（含缓冲）
- [ ] 成本上限是否明确？

## 执行阶段检查

- [ ] 是否搭建了项目骨架后再填充功能？（框架先行）
- [ ] 每个模块完成后是否立即测试？  ⬅️ 重要！
- [ ] 看板是否每日更新？  ⬅️ 本项目教训（未做到）
- [ ] 遇到阻塞是否立即记录并评估 Plan B？

## 监控阶段检查

- [ ] 是否定期（至少每 3 天）做偏差分析？
- [ ] 偏差 > 10% 是否立即预警？
- [ ] 调整方案是否提供了至少 2 个选项？
- [ ] 决策是否有记录？

## 验收阶段检查

- [ ] 是否逐项对照原始 SMART 目标验收？
- [ ] 完成率是否量化？
- [ ] 成功经验是否提炼了 ≥ 3 条？（含具体场景）
- [ ] 失败教训是否提炼了 ≥ 3 条？（含改进建议）
- [ ] 意外事件是否记录？
- [ ] 所有文档是否归档？
- [ ] 后续演进方向是否明确？

## 本项目关键经验速查

| 编号 | 经验 | 适用场景 |
|------|------|----------|
| E1 | 框架先行：先搭骨架再填充 | 所有 CLI/Web 项目 |
| E2 | 增量设计：同步记录水位线 | 涉及外部同步的功能 |
| E3 | Plan B 前置：策略阶段就定降级方案 | 有外部依赖的功能 |
| E4 | 依赖验证：资源阶段实际测试可用性 | 第三方 API/凭证 |
| E5 | 测试即时：完成即测，不攒到最后 | 所有开发工作 |
