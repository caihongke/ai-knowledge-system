> 基于现有七步法框架，整合编剧双赛道创作流程

## 一、现状分析

### 1.1 现有架构能力
- **分析类Agent**: step1-diag ~ step3-strategy（规划层）
- **执行类Agent**: step4-resource ~ step5-execute（执行层）

- **知识库**: 笔记管理、间隔复习、飞书同步

### 1.2 编剧系统需求
- **双赛道模式**: 短视频剧本(3-5min) vs 网文长篇(100万字+)
- **工业化组件**: 人设/桥段/爽点模块化
- **AI风控体系**: 五大铁律硬性约束
- **数据沉淀**: 创作-拉片-迭代闭环

### 2.3 五大铁律风控系统

```python
# core/script_guard.py
class ScriptGuard:
    """AI风控校验系统 - 五大铁律"""

    IRON_RULES = {
        "IR-001": {
            "name": "价值观安全",
            "check": lambda text: check_value_safety(text),
            "action": "block",  # block/warn/log
            "auto_fix": False
        },
        "IR-002": {
            "name": "版权合规",
            "check": lambda text: check_copyright_risk(text),
            "action": "block",
            "auto_fix": False
        },
        "IR-003": {
            "name": "平台规则",
            "check": lambda text, platform: check_platform_rules(text, platform),
            "action": "warn",
            "auto_fix": True
        },
        "IR-004": {
            "name": "逻辑自洽",
            "check": lambda context: check_logic_consistency(context),
            "action": "warn",
            "auto_fix": False
        },
        "IR-005": {
            "name": "人设稳定",
            "check": lambda character, actions: check_character_consistency(character, actions),
            "action": "block",
            "auto_fix": False
        }
    }

    def validate(self, content: dict, track: str) -> ValidationResult:
        """
        执行五大铁律校验
        - track: "short" | "long"
        - 触碰铁律 → 强制人工整改
        """
        violations = []
        for rule_id, rule in self.IRON_RULES.items():
            result = rule["check"](content)
            if not result.passed:
                violations.append({
                    "rule": rule_id,
                    "name": rule["name"],
                    "action": rule["action"],
                    "details": result.details
                })
                if rule["action"] == "block":
                    return ValidationResult(
                        passed=False,
                        blocked=True,
                        violations=violations,
                        message=f"触碰铁律 {rule_id}: {rule['name']}，必须人工整改"
                    )

        return ValidationResult(passed=True, violations=violations)
```

```python
# creation/component_engine.py
class ComponentEngine:
    """工业化组件引擎 - 按需开启"""

    def __init__(self):
        self.active_components = set()
        self.track = None  # short | long

    def load_component(self, component_id: str, track: str):
        """
        按需加载组件，避免全量堆叠
        """
        if track != self.track:
            self.clear_all()
            self.track = track

        # 检查组件兼容性
        if not self._check_compatibility(component_id):
            raise ComponentConflictError(f"{component_id} 与当前组件冲突")
```

### 4.2 迭代优化Agent

```python
# creation/agents/iteration_engine.py
class IterationEngine:
    """迭代优化引擎"""

    def analyze_gap(self, creation: CreationSession) -> GapAnalysis:
        """
        分析创作与目标的差距
        """
        target = self._get_target_metrics(creation.track)
        actual = self._extract_actual_metrics(creation)

        return GapAnalysis(
            dimensions=["hook", "conflict", "emotion", "structure"],
            gaps={k: target[k] - actual[k] for k in target.keys()},
            priority=sorted(gaps.items(), key=lambda x: -x[1])
        )

    def generate_improvement_plan(self, gap: GapAnalysis) -> ImprovementPlan:
        """
        生成改进方案
        """
        # 基于差距选择优化策略
        strategies = []
        for dim, delta in gap.priority[:3]:
            strategies.append(self._get_strategy(dim, delta))

        return ImprovementPlan(
            strategies=strategies,
            expected_improvement=self._estimate_improvement(strategies),
            components_needed=self._identify_components(strategies)
        )
```

## 七、人机协同工作流

### 7.1 标准创作流程

```
用户输入需求
    ↓
[step1-diag] 诊断赛道 + 需求确认
    ↓
[script-short/long] 生成初版大纲
    ↓
[ScriptGuard] 五大铁律校验
    ↓ (通过) / (不通过)→人工整改
[step2-blueprint] 系统蓝图细化
    ↓
用户确认/修改
    ↓
[step3-strategy] 生成详细剧情
    ↓
[ComponentEngine] 按需加载组件
    ↓
用户迭代创作
    ↓
[ScriptGuard] 阶段自检
    ↓
[step5-execute] 成品输出
    ↓
[story-analyzer] 拉片分析
    ↓
[iteration-engine] 优化建议
    ↓
数据沉淀至知识库
```

### 7.2 关键决策点

| 决策点 | 人工决策 | AI辅助 |
|--------|----------|--------|
| 赛道选择 | ✓ 必须 | 提供建议 |
| 创意方向 | ✓ 必须 | 生成选项 |
| 铁律校验 | ✗ 不可修改 | 硬性执行 |
| 组件选择 | ✓ 按需开启 | 推荐组合 |
| 迭代优化 | ✓ 确认执行 | 生成方案 |

## 九、总结

本优化方案将编剧双赛道创作流程完整整合进现有七步法多智能体架构：

1. **新增创作类Agent** - script-short、script-long、story-analyzer、iteration-engine
2. **强化风控体系** - ScriptGuard 实现五大铁律硬性约束
3. **工业化组件** - ComponentEngine 支持按需加载，避免创作僵硬
4. **数据闭环** - 创作-拉片-迭代完整链路，数据沉淀至知识库
5. **人机协同** - 明确人工决策点与AI自动执行边界

预计实施周期：**7周**，与编剧流程Day1-7一一对应。
